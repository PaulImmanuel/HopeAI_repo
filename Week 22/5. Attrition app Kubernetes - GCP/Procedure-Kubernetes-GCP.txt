Step4: Go to IAM from left side three stacked horizontal line

Stpe9: Install the downloaded .exe file.

Step10: Download Docker SDK

https://docs.docker.com/desktop/install/windows-install/

Step11: Install the downloaded file 

Step12: After installation, open --> Command Prompt

Step15: Download and install 

https://kubernetes.io/releases/download/#binaries

Step16:Goto command prompt

kubectl version --client

gcloud auth login

gcloud init


Use the GC proj ID:
gcloud config set project attrition-kubernetes-gcp


gcloud services enable cloudbuild.googleapis.com


Use your Docker Username: [dockerusername/any-app-name .]
docker build -t paulimmanuel/attrition_app .


Use format: [dockerusername/dockerappName gcr.io/gc proj id/dockerappName]
docker tag paulimmanuel/attrition_app gcr.io/attrition-kubernetes-gcp/attrition_app


gcloud auth configure-docker

docker push gcr.io/attrition-kubernetes-gcp/attrition_app


gcloud services enable container.googleapis.com

gcloud components install gke-gcloud-auth-plugin

if error happens:
open "google cloud sdk shell" --> run as a adminstartor-->gcloud components install gke-gcloud-auth-plugin

gcloud container clusters create attrition-kubernetes-gcp --zone us-central1-a --num-nodes=1 (-hypens mmust)

gcloud container clusters get-credentials attrition-kubernetes-gcp --zone us-central1-a

gcloud components update

Create a file "attrition-deployment.yaml" in your project directory (the path where you have opened cmd now)

apiVersion: apps/v1
kind: Deployment
metadata:
  name: attrition-app-deployment
spec:
  replicas: 1
  selector:
    matchLabels:
      app: attrition-app
  template:
    metadata:
      labels:
        app: attrition-app
    spec:
      containers:
      - name: attrition-app
        image: gcr.io/attrition-kubernetes-gcp/attrition_app
        ports:
        - containerPort: 8080  # Change to 8080 if that is your app's listening port


kubectl apply -f attrition-deployment.yaml

_________________________

Create a file "attrition-service.yaml" in your project directory (the path where you have opened cmd now)

apiVersion: v1
kind: Service
metadata:
  name: attrition-app-service
spec:
  selector:
    app: attrition-app
  ports:
    - protocol: TCP
      port: 80         # External Port that users will access
      targetPort: 8080 # The port your app is running on inside the container
  type: LoadBalancer

kubectl apply -f attrition-service.yaml

kubectl get svc

go to GCP kubernetes engine -->gatesways,service and ingress


gcloud container clusters list --project attrition-kubernetes-gcp

gcloud container clusters delete attrition-kubernetes-gcp  --zone us-central1-a --project attrition-kubernetes-gcp

gcloud container clusters delete attrition-app-service --zone us-central1-a
